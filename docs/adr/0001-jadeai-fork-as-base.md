# ADR-0001：以 JadeAI 为主工程

状态：Accepted
日期：2026-07-16

## 背景

候选项目各自覆盖部分能力，但没有单一项目同时具备 JadeAI 的模板、编辑器、导出、AI 和
面试基础，以及目标需求中的账号、GitHub 知识库和证据驱动生成。

## 决策

Fork JadeAI 最新稳定上游作为主工程，保留 Apache-2.0 许可证要求。其他项目只用于：

- 参考交互和领域设计。
- 在许可证允许且经过评审时移植小型独立模块。
- 默认在 TypeScript/Next.js 技术栈中重新实现。

`sub2api` 的账号设计采用 Clean-room 方式借鉴，不直接复制 LGPL-3.0 Go/Vue 代码。
许可证不明确的 `markdown-to-resume` 不复制代码、模板和资源。

## 后果

- 保留现有简历功能并降低迁移成本。
- 必须系统性修复上游认证、租户和 AI 写回边界。
- 上游更新通过 `upstream` Remote 定期评估，不自动合并生产。
- 本机生产目录与开发目录严格分离。
